About this predecessor work

This is the earlier SCHWAN-project publication that established the ICEP-R automated coding task: predicting Infant and Caregiver Engagement Phases–Revised (ICEP-R) labels from frozen audio-visual foundation-model features (DINOv2 for vision, Wav2Vec2-BERT for audio) using lightweight downstream classifiers - a bidirectional LSTM for the full temporal engagement task and linear classifiers for unimodal encodings. The pipeline integrates with the NOVA / DISCOVER annotation ecosystem developed at the Human-Centered Multimedia lab (hcmlab).

This year's ICMI '26 publication builds directly on this work: it reuses the same ICEP-R task, corpus, and frozen-feature baseline family (now with DINOv3 + Wav2Vec 2.0) as a reference point, and extends it with zero-shot and LoRA-fine-tuned Multimodal LLMs (Qwen2.5-Omni, Qwen2-Audio, Qwen3-VL, GLM-4.6), which substantially outperform the frozen-feature approach introduced here.

Code

The original implementation lives in its own repository and is linked here rather than duplicated: github.com/Daksitha/SCHWAN-ICEP-R-Automation.

  • feature_extraction/ - DINOv2 (vision) and Wav2Vec2-BERT (audio) feature extraction
  • training/ - bidirectional LSTM (temporal engagement) and linear classifiers (unimodal encodings)
  • data_processing/ - dataset construction and visualisation utilities
This repository predates the ICMI '26 release and is maintained independently; see its own README for setup instructions and requirements.
